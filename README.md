# RUthirsty - 喝水打卡应用

一个简洁实用的喝水打卡Cordova应用，帮助你养成良好的喝水习惯。

## 功能特性

### 核心功能
- ✅ **一键打卡** - 大按钮设计，轻松记录每次喝水
- ✅ **快速打卡** - 一次性添加2/3/5条记录，补录更方便
- ✅ **今日统计** - 实时显示今日喝水次数和完成进度
- ✅ **目标设置** - 自定义每日喝水目标（1-20杯）
- ✅ **进度可视化** - 动态颜色进度条，完成度一目了然
- ✅ **记录管理** - 查看今日记录，支持删除误操作
- ✅ **历史查看** - 按日期分组查看所有历史记录

### 高级功能
- 🎉 **完成祝贺** - 达到每日目标时显示动画祝贺
- 📊 **总体统计** - 显示总打卡次数、连续打卡天数、累计天数
- 💾 **数据导出** - 导出CSV格式数据，可用Excel打开
- 🗑️ **数据管理** - 清空所有数据功能（双重确认保护）
- 🎨 **智能配色** - 进度条根据完成度自动变色
- 💬 **即时反馈** - Toast提示消息，操作反馈更友好

### 界面特色
- 🎨 **简洁界面** - 清新的紫色渐变主题，简洁实用
- 📱 **响应式设计** - 完美适配手机、平板等各种屏幕
- ⚡ **流畅动画** - 精心设计的过渡动画和交互效果
- 💾 **数据持久化** - 使用localStorage本地存储，数据不丢失

## 快速开始

### 在浏览器中测试

1. 启动本地开发服务器：
```bash
cd www
python3 -m http.server 8000
```

2. 在浏览器中访问：`http://localhost:8000`

### 构建Android应用

#### 前置要求

- Node.js 和 npm
- Cordova CLI (`npm install -g cordova`)
- Android SDK（需要设置 ANDROID_HOME 环境变量）
- Java JDK

#### 构建步骤

1. 安装依赖：
```bash
npm install
```

2. 添加Android平台（如果还没有）：
```bash
cordova platform add android
```

3. 构建APK：
```bash
cordova build android
```

4. 运行到设备或模拟器：
```bash
cordova run android
```

生成的APK文件位于：`platforms/android/app/build/outputs/apk/`

## 使用说明

### 基础操作

#### 打卡喝水
- **单次打卡**：点击中央的大圆形"打卡喝水"按钮，即可记录一次喝水
- **快速打卡**：使用下方的 +2、+3、+5 按钮，一次性添加多条记录（适合补录）
- 按钮会有动画反馈，记录会立即显示在下方列表中

#### 设置目标
1. 点击右上角的"设置"按钮
2. 输入每日目标杯数（1-20）
3. 点击"保存"

**进度条颜色说明：**
- 🔘 灰色：完成度 < 50%
- 🔵 蓝色：完成度 50-75%
- 🟣 紫色：完成度 75-100%
- 🟢 绿色：已完成 100%

#### 查看记录
- **今日记录**：主界面下方显示今天的所有打卡记录，按时间倒序排列
- **历史记录**：点击"查看历史记录"按钮，可以查看所有历史数据，按日期分组显示
- **总体统计**：查看总打卡次数、连续打卡天数、累计打卡天数

#### 删除记录
每条记录右侧都有"删除"按钮，点击后确认即可删除该记录。

### 高级功能

#### 数据导出
1. 点击"导出数据"按钮
2. 系统会自动生成CSV文件并下载
3. 文件名格式：`RUthirsty_喝水记录_2026-02-07.csv`
4. 可用Excel、Numbers等软件打开查看

**导出数据格式：**
```csv
日期,时间,时间戳
2026-02-07,14:30:25,1707307825000
2026-02-07,11:15:42,1707296142000
```

#### 清空数据
1. 点击"设置"按钮
2. 滚动到底部的"危险操作"区域
3. 点击"清空所有数据"按钮
4. 需要两次确认才能执行（防止误操作）
5. **建议先导出数据备份**

#### 完成目标祝贺
当你达到每日目标时，应用会自动显示祝贺动画，鼓励你继续保持良好习惯！

## 注意事项

### Android SDK 配置

如果构建Android应用时遇到 "ANDROID_HOME not found" 错误，需要：

1. 安装 Android SDK
2. 设置环境变量：
```bash
export ANDROID_HOME=/path/to/android-sdk
export PATH=$PATH:$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools
```

### 在GitHub Codespaces中

当前开发环境配置了Android SDK feature，但可能需要重新构建容器才能使用。如果无法构建Android应用，可以：

1. 在浏览器中测试所有功能
2. 在本地环境中构建Android应用
3. 或使用在线构建服务（如 PhoneGap Build）
